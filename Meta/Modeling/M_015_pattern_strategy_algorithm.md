# M-ALG-001  
## 算法 / 策略 / 模式 —— 在建模链路中的精确区分

---

## 0. 核心结论（定锚）

**模式定义解法的结构空间，  
策略定义在该结构中如何做决策，  
算法则把策略落实为可执行的计算过程。**

三者不在同一抽象层级，不能混用。

---

## 1. 增强版建模链路

原始链路：

现实问题
→ 数学模型
→ 计算模型
→ 算法
→ 语言实现


精确分层后：

现实问题
→ 数学模型
→ 计算模型
→ 算法模式（Pattern）
→ 算法策略（Strategy）
→ 算法（Algorithm）
→ 语言实现


这是一个**抽象逐级降低**的链路，而不是同义替换。

---

## 2. 各层职责边界

### 2.1 数学模型（What）

回答：**问题是什么**

- 对象 / 集合
- 约束条件
- 目标函数（min / max / feasibility）
- 与“怎么求”无关

例：  
区间集合 + 不重叠约束 + 最大化区间数量

---

### 2.2 计算模型（What is computable）

回答：**允许怎样的计算**

- 顺序 / 并行
- 是否允许随机访问
- 是否允许记忆（DP 前提）
- 状态是否可枚举

例：
- RAM 模型
- 图是显式还是隐式
- 状态空间大小是否可控

---

## 3. 算法模式（Pattern）——结构层

### 3.1 定义

**算法模式是对一类问题“可行解结构”的经验性抽象。**

回答：
> 这类问题通常如何组织解法？

### 3.2 常见模式

- 贪心模式
- 动态规划模式
- 分治模式
- 回溯 / 搜索模式
- 图遍历模式

### 3.3 本质特征

- 不是算法
- 不保证正确
- 提供思考框架

> 模式 ≈ 解法空间的“坐标系”

---

## 4. 算法策略（Strategy）——决策层

### 4.1 定义

**策略是：在某一算法模式下采用的具体决策原则。**

回答：
> 在关键决策点，按什么标准选？

### 4.2 策略依赖数学结构

- 不同目标函数 → 不同策略
- 同一模式下可有多种策略

例（贪心模式）：
- 最早结束时间
- 最大权重
- 最小边权
- 最大单位价值

### 4.3 正确性的核心

- 贪心是否成立 ≠ 模式是否合理
- 而是：**该策略是否满足贪心选择性质**

---

## 5. 算法（Algorithm）——执行层

### 5.1 定义

**算法是策略在给定计算模型下的具体可执行实现。**

回答：
> 如何用有限步骤完成计算？

### 5.2 必须明确的要素

- 数据结构
- 控制流
- 初始化 / 终止条件
- 时间 / 空间复杂度

### 5.3 重要区分

- 一个策略 ≠ 一个算法  
- 同一策略可有多种算法实现

---

## 6. 贯穿示例：区间调度问题

### 数学模型
- 区间集合
- 不重叠约束
- 最大化区间数量

### 计算模型
- 排序 + 顺序扫描
- RAM 模型

### 算法模式
- 贪心模式

### 算法策略
- 每次选择结束时间最早的区间

### 算法
1. 按结束时间排序
2. 初始化当前结束点
3. 顺序扫描
4. 可选则加入解集

### 实现
- Python / C++ / Java

---

## 7. 常见误区（反例意识）

错误说法：
> “贪心算法不对”

正确表述：
- 贪心**模式**是合理的
- 但当前**策略**不满足贪心选择性质
- 因此不存在保证最优的**算法**

---

## 8. 万能论述句（KT / 面试可用）

> 我先分析问题的数学结构，判断是否存在最优子结构；  
> 在此基础上选择合适的算法模式；  
> 然后提出具体策略并通过交换论证验证其正确性；  
> 最后给出对应的可执行算法与复杂度分析。

---

## 9. 层级速览表

| 层级 | 名称 | 关注点 | 是否可执行 |
|----|----|----|----|
| 高 | 模式 | 解法结构 | 否 |
| 中 | 策略 | 决策规则 | 否 |
| 低 | 算法 | 执行步骤 | 是 |
