# 建模总纲：Problem-Solving Model × System Model

> 本文是整个建模体系的**基础总纲**，用于澄清两类根本不同但彼此正交的建模问题：
>
> - **我们要解决的问题，如何一步步走向实现？**
> - **我们所研究或构建的系统，本身是什么样的？**
>
> 为避免概念混淆，本文明确区分：
> - **问题解决模型（Problem-Solving Model）**
> - **系统模型（System Model）**

---

## 0. 核心立场（Core Thesis）

> **建模至少存在两个正交维度：**
>
> - 一个维度关心 **How**：问题如何被解决  
> - 一个维度关心 **What**：系统本身是什么  
>
> **Agent 属于系统模型；  
> “问题 → 实现”的纵向建模属于问题解决模型。**  
>
> 二者不构成上下级关系，也不能相互替代。

---

## 一、问题解决模型（Problem-Solving Model）

### 1.1 定义

**Problem-Solving Model** 描述的是：

> 如何将一个现实问题，  
> 逐层转化为可计算、可验证、可实现的解决方案。

它是一种**方法论模型**，而不是系统本体描述。

---

### 1.2 标准纵向建模链路

现实问题
→ 数学模型
→ 计算模型
→ 算法
→ 语言实现


这不是“同一件事的不同表述”，  
而是**连续的、不可跳过的再建模过程**。

---

### 1.3 各层级的角色

#### 现实问题
- 来源于真实世界
- 语义模糊
- 约束不完全显式
- 通常不可直接计算

---

#### 数学模型
- 对现实问题的**结构性抽象**
- 明确定义：
  - 对象
  - 关系
  - 状态
  - 动态
- 目标是**逻辑推演与结构分析**

> 数学模型关注“必然性”，而非执行性。

---

#### 计算模型
- 引入可计算性假设
- 明确：
  - 时间模型
  - 空间 / 内存模型
  - 并行性假设
- 典型例子：RAM 模型、并行计算模型

---

#### 算法
- 在计算模型约束下的**执行策略**
- 不改变问题语义
- 改变的是：
  - 时间复杂度
  - 空间复杂度
  - 稳定性与精度权衡

---

#### 语言实现
- 将算法映射到具体编程语言与系统
- 引入：
  - 类型系统
  - 运行时
  - 工程约束
- 属于**工程落地层**，不再是建模抽象

---

### 1.4 方法论总结

> **Problem-Solving Model 关注的是 How：  
> 我们如何一步步把问题变成可执行方案。**

---

## 二、系统模型（System Model）

### 2.1 定义

**System Model** 描述的是：

> 系统本身由什么组成，  
> 各组成部分如何相互作用，  
> 整体行为如何产生。

它属于**对象论 / 本体论层面**。

---

### 2.2 系统模型的核心要素

一个系统模型通常需要明确：

- 基本单元（Components）
- 状态空间（State）
- 行为规则（Behavior）
- 交互机制（Interaction）
- 演化方式（Dynamics）

---

### 2.3 Agent 模型的准确定位

> **Agent 模型是一种系统模型。**

在 Agent-based System 中：

- 系统由多个 Agent 构成
- 每个 Agent：
  - 具有局部状态
  - 遵循行为规则
  - 基于局部信息进行决策
- 系统宏观行为由 Agent 交互**涌现**

---

### 2.4 重要澄清（避免混乱）

- Agent ≠ 算法  
- Agent ≠ 编程范式  
- Agent ≠ 问题解决流程  

Agent 是一种**系统结构假设**，  
而不是“如何实现”的说明。

---

### 2.5 常见系统模型类型（示例）

- Agent-based system
- Statistical / Data-driven system
- Continuous dynamical system
- Rule-based system
- Hybrid system
- Cyber-physical system

这些模型之间：
- 不存在层级高低
- 只存在**对问题本质的不同假设**

---

### 2.6 系统模型总结

> **System Model 关注的是 What：  
> 系统是什么、如何运作。**

---

## 三、两个模型的关系（关键结论）

### 3.1 正交关系

> **系统模型决定：我们假定世界“长什么样”  
> 问题解决模型决定：我们如何对这个世界进行求解与实现**

二者是**正交维度**：

       System Model（What）
            ↑
Agent / Data / Hybrid / Rule-based
↑
            ↓
 Problem-Solving Model（How）
问题 → 数学 → 计算 → 算法 → 实现
