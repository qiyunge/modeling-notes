# Algorithm Design – Paradigm Decision Tree
# 算法范式决策树（方法论级）

> 目标：  
> 在面对一个新问题时，用最少的试错，快速判断「该用哪一类算法思想」。

---

## 总入口问题（Root Question）

### Q0：这个问题是在 **求最优解** 吗？
- ❌ 否 → 可能是：
  - 模拟 / 规则
  - 数据结构设计
  - 在线算法 / 系统问题  
  → 不走本树
- ✅ 是 → 继续

---

## 第一层：动态规划（DP）

### Q1：问题是否具备 **清晰的最优子结构**？
- 子问题可以独立定义
- 子问题解可组合为全局解
- 决策不会产生强全局耦合

#### ✅ YES → 用 DP
- 定义状态
- 写转移
- 控制维度

#### ❌ NO → 进入 Greedy 探测

📌 失败信号：
- 状态必须“记住太多历史”
- 一个决策会影响所有未来路径

---

## 第二层：贪心（Greedy as Probe）

### Q2：是否存在一个 **自然的局部最优选择规则**？
- 每一步都有一个“看起来最合理”的选择
- 不需要回溯？

#### ✅ YES → 尝试 Greedy
- 并 **刻意寻找反例**

##### Q2.1：Greedy 是否总是成功？
- ✅ YES → Greedy 正确
- ❌ NO → 记录失败模式

#### ❌ NO → 直接进入结构层（Graph / Search）

📌 Greedy 的真正作用：
> 不是给答案，而是暴露结构

---

## 第三层：失败分析 → 结构抽象

### Q3：Greedy 失败是因为什么？
- ❓ 早期决策阻断了未来可能？
- ❓ 需要“撤销 / 回退”？
- ❓ 局部选择之间存在冲突？

👉 如果 YES → 问题是 **结构耦合问题**

---

## 第四层：图模型（Graph Modeling）

### Q4：问题是否可以表示为：
- 节点 = 状态 / 对象
- 边 = 关系 / 约束 / 转移
- 权重 / 容量 = 限制？

#### ✅ YES → 转成图问题

常见映射：
- 路径 → 最短路
- 资源流动 → 最大流
- 冲突匹配 → 匹配
- 可达性 → DFS / BFS

#### ❌ NO → 考虑搜索 / 数学建模

---

## 第五层：图中的结构级算法

### Q5：图问题属于哪一类？

#### a) 是否是“资源在网络中流动”？
→ **Maximum Flow / Min Cut**

#### b) 是否是“一对一 / 一对多匹配”？
→ **Bipartite Matching / Assignment**

#### c) 是否是“可达性 / 层次”？
→ **BFS / DFS / Topological Sort**

#### d) 是否是“强耦合子结构”？
→ **SCC / Condensation Graph**

---

## 第六层：分治 / 搜索（兜底结构）

### Q6：是否可以：
- 拆成子问题？
- 通过搜索枚举？
- 通过剪枝降低复杂度？

常见方法：
- Divide & Conquer
- Backtracking
- DFS / BFS State Space
- Branch & Bound

📌 特点：
> 正确但可能慢  
> 为后续优化打基础

---

## 第七层：数学建模（LP / ILP）

### Q7：是否可以形式化为：
- Linear Programming
- Integer Constraints
- 对偶结构？

#### ✅ YES →
- 看是否有整数解
- 看是否存在组合算法等价形式

📌 很多“高级算法”的理论根源在这里

---

## 第八层：近似 / 随机化 / 启发式（现实层）

### Q8：问题是否：
- NP-hard？
- 规模巨大？
- 工程时限严格？

→ 使用：
- Approximation Algorithms
- Randomized Algorithms
- Heuristics
- Metaheuristics（GA / SA / RL）

⚠️ 这是“现实决策”，不是“理论失败”

---

## 终极总结（必须记住）

> 算法设计 ≠ 套模板  
> 算法设计 = 一连串 **理性否定**

### 正确思路顺序：
1. 能 DP 就 DP
2. DP 不自然 → 用 Greedy 试探
3. Greedy 失败 → 分析失败结构
4. 结构问题 → 图模型
5. 图 → 流 / 匹配 / 割
6. 不行 → 搜索 / 数学建模
7. 最后才是近似

---

## 一句话箴言

> **不要问“该用什么算法”，  
> 要问“这个问题拒绝了哪些算法”。**
